# 志向確認ヒアリング

## 目的

- 今後社員を増やし、会社を拡大するための（可能かどうかも含めた）情報を取得すること
- 管理職候補となる方のピックアップ、退職リスクのフォロー、社員の志向の確認

### 離職リスクとして特に確認しておきたい項目

※今回のヒアリングですべてを把握できなくても可

| 観点 |
| --- |
| 年齢 |
| 金銭満足度 |
| ポジション満足度 |
| スキルアップ満足度 |
| 仕事満足度 |
| 居心地満足度 |
| 残業 |
| よそよそしさ |
| 評価納得度 |

## ヒアリング内容

### 1. 志向に関する確認

- 現在の業務、役割は？満足していますか？（金銭的、ポジション的、仕事的、居心地的、スキルアップ的）
- 今後やってみたいことはありますか？※特になくてもよい
- 将来どのようになっていたいですか？（エンジニア、リーダー、マネージャー、フルスタックエンジニア、特に考えていない）

### 2. 変化に対する耐性を測る

- 今の会社の良いところは何だと思いますか？
- 逆に改善してほしいと思っているところ、会社への要望はありますか？
- AIツールを今の業務でどう使っていますか？／使ってみたいですか？

### 3. 不安解消目的

- 他に何か気になっていることや不安なことはありますか？

## 最終的に策定していくもの

### A. 組織診断

会社拡大の可否を2つの問いで診断する。各問いはルブリックの特定スコアに紐づく。

| 問い | 意味 | 推進材料（プラス要因） | ボトルネック（マイナス要因） |
| --- | --- | --- | --- |
| **拡大できるか** | 人員を増やしたときに既存社員が新メンバー・新業務を受け入れられるか／拡大した組織を束ねる管理職レイヤーを社内で作れるか | （後で再検討） | （後で再検討） |
| **変化できるか** | PMI・制度変更・組織再編といった構造変化、およびAI活用シフトなどの技術・業務内容変化に耐えられるか | （後で再検討） | （後で再検討） |

2つとも揃って初めて「拡大可能」。欠けている軸への打ち手（採用・育成・処遇）をPMI戦略に組み込む。

### B. 人材マップ（3軸構造）

各社員を以下の3軸で記述する。詳細な分類定義と採点基準は [`評価ルブリック.md`](./評価ルブリック.md) を参照。

- **軸A 貢献タイプ**：中核人材／管理候補／スペシャリスト／変革推進役／一般戦力（標準層）／判定保留
- **軸B 動機タイプ**：成長志向／承認・地位志向／貢献・使命志向／安定志向／生活重視
- **軸C 離職リスク**：1〜5スコア＋5フラグ（軸Aとの掛け合わせで対処優先度を決定）

### C. PMI戦略

組織診断の結果を踏まえ、拡大・変革をどう進めるかを4つの意思決定に分解する。

| 意思決定 | 論点 | 判定材料 | 典型的な分岐 |
| --- | --- | --- | --- |
| **拡大のスピード感** | 一気に増やすか、段階的に増やすか | （後で再検討） | （後で再検討） |
| **まず安定化するか** | 拡大・変革に着手する前に既存組織を固めるか | （後で再検討） | （後で再検討） |
| **制度変更どこまでやるか** | 評価・報酬・組織構造の制度をどこまで触るか | （後で再検討） | （後で再検討） |
| **誰を先に巻き込むか** | 変革の初動メンバー（旗振り役・巻き込み順序）を誰にするか | （後で再検討） | （後で再検討） |

※4つは独立ではない。安定化フェーズの長さが拡大スピードを規定し、制度変更の幅が巻き込み順序の難度を上げる。**「まず安定化」と「制度変更の幅」を先に決めると残り2つが自動的に絞られる**。
